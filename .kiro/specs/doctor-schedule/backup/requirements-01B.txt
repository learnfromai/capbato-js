Page Display and Navigation:
WHEN a user navigates to the doctor-schedule page 
THE SYSTEM SHALL display the page header with clinic logo, user role, username, and profile dropdown

WHEN the doctor-schedule page loads 
THE SYSTEM SHALL display the sidebar navigation with "Doctors" menu item highlighted as active

WHEN the doctor-schedule page is displayed 
THE SYSTEM SHALL show two main sections: doctors table and calendar view with "Doctor's Schedule" title

---

Authentication and User Profile:
WHEN the doctor-schedule page loads 
THE SYSTEM SHALL retrieve and display the logged-in user's role and username from localStorage

WHEN the logged-in user role is "admin" 
THE SYSTEM SHALL add an "Accounts" menu item to the sidebar navigation

WHEN a user clicks on the profile avatar 
THE SYSTEM SHALL display a dropdown menu with "Settings" and "Logout" options

WHEN a user clicks "Logout" from the profile dropdown 
THE SYSTEM SHALL clear localStorage data and redirect to login.html

---

Doctors Table Display and Data Loading:
WHEN the doctor-schedule page loads 
THE SYSTEM SHALL fetch doctors data from the /doctors API endpoint

WHEN the doctors API call succeeds 
THE SYSTEM SHALL display doctors in a table with columns: Doctor's Name, Specialization, Contact Number

WHEN the doctors API call fails 
THE SYSTEM SHALL display fallback doctor data with sample doctors (Maria Santos, Juan Cruz, Anna Reyes)

WHEN rendering doctors in the table 
THE SYSTEM SHALL format names as "Dr. [FirstName] [LastName]" and show "N/A" for missing contact numbers

---

Calendar Display and Schedule Loading:
WHEN the doctor-schedule page loads 
THE SYSTEM SHALL fetch schedule data from the /schedules API endpoint and display current month calendar 

WHEN the calendar displays 
THE SYSTEM SHALL show a monthly grid with day headers (Sun, Mon, Tue, Wed, Thu, Fri, Sat)

WHEN the calendar renders 
THE SYSTEM SHALL display dates with schedule information showing doctor name and time in blue text

WHEN a date has no scheduled doctor 
THE SYSTEM SHALL display only the date number without additional information

---

Calendar Navigation:
WHEN a user clicks the previous month button (left chevron) 
THE SYSTEM SHALL navigate to the previous month and update the calendar display

WHEN a user clicks the next month button (right chevron) 
THE SYSTEM SHALL navigate to the next month and update the calendar display

WHEN the month changes 
THE SYSTEM SHALL update the month label to show the new month and year in uppercase format

WHEN navigating between months 
THE SYSTEM SHALL maintain the loaded schedule data and display relevant schedules for the viewed month

---

Edit Schedule Functionality:
WHEN a user clicks the "Edit" button 
THE SYSTEM SHALL display an overlay with an iframe loading edit-sched.html

WHEN the edit schedule overlay opens 
THE SYSTEM SHALL display a close button (X) in the top-right corner of the overlay

WHEN a user clicks the close button 
THE SYSTEM SHALL hide the overlay and clear the iframe source

WHEN the edit schedule form is submitted successfully 
THE SYSTEM SHALL close the overlay and reload the page to show updated schedule data

---

Edit Schedule Form Handling:
WHEN the edit schedule form loads 
THE SYSTEM SHALL display input fields for Doctor Name, Date, and Time with appropriate labels

WHEN the edit schedule form is displayed 
THE SYSTEM SHALL require all fields (doctor name, date, time) to be filled before submission

WHEN a user submits the edit schedule form 
THE SYSTEM SHALL send a POST request to /schedules endpoint with doctor, date, and time data

WHEN the schedule submission succeeds 
THE SYSTEM SHALL send a "scheduleUpdated" message to the parent window to trigger page reload

---

Sidebar Navigation:
WHEN a user clicks "Dashboard" in the sidebar 
THE SYSTEM SHALL navigate to index.html

WHEN a user clicks "Appointments" in the sidebar 
THE SYSTEM SHALL navigate to appointments.html

WHEN a user clicks "Patients" in the sidebar 
THE SYSTEM SHALL navigate to patients.html

WHEN a user clicks "Laboratory" in the sidebar 
THE SYSTEM SHALL navigate to laboratory.html

WHEN a user clicks "Prescriptions" in the sidebar 
THE SYSTEM SHALL navigate to prescriptions.html

WHEN a user clicks "Doctors" in the sidebar 
THE SYSTEM SHALL navigate to doctor-schedule.html (current page)

---

Error Handling and Fallback:
WHEN the /schedules API call fails 
THE SYSTEM SHALL log the error to console and continue displaying calendar without schedule data

WHEN the /doctors API call fails 
THE SYSTEM SHALL log the error to console and display fallback doctor data

WHEN localStorage lacks user authentication data 
THE SYSTEM SHALL still display the page but may show empty role/username fields

WHEN the edit schedule iframe fails to load 
THE SYSTEM SHALL still display the overlay container with close functionality

---

Visual Feedback and User Experience:
WHEN hovering over navigation menu items 
THE SYSTEM SHALL change background color to light gray and text color to teal

WHEN hovering over the profile avatar 
THE SYSTEM SHALL increase scale to 1.05 and enhance shadow for visual feedback

WHEN hovering over calendar cells with schedules 
THE SYSTEM SHALL maintain the default styling without additional hover effects

WHEN the current navigation item is active 
THE SYSTEM SHALL display green background, teal text, and left border indicator
